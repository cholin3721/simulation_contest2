# (주)K-Logistics 시뮬레이션 모델링 물리 & 논리 규칙 (Rules)

본 문서는 문제지 도면(<그림 1>)과 명세서의 논리적 충돌을 방지하고, 물리적 레이아웃과 AnyLogic 객체 간의 일관성을 유지하기 위해 정의된 프로젝트 내부 규칙입니다.

## 1. '스테이션(Station)'과 '랙(Rack)', '통로(Aisle)'의 상관관계 정의

도면의 시각적 형태(회색 랙, 통로)와 문제지 텍스트(스테이션 개수)를 교차 검증한 결과, 본 프로젝트에서는 **"1 스테이션 = 1 랙(단일 선반)"**으로 엄격하게 정의합니다. 랙을 나란히 배치할 때 자연스럽게 형성되는 공간이 통로(Aisle)이며, 개수 공식은 `통로 개수 = 랙 개수 - 1` 입니다.

*   **S급 구역:** 스테이션 **3개** = 선반(Rack) **3동** = 작업 통로 **2개**
*   **A급 구역:** 스테이션 **7개** = 선반(Rack) **7동** = 작업 통로 **6개**
*   **B급 구역:** 스테이션 **4개** = 선반(Rack) **4동** = 작업 통로 **3개**

*(주의: 기존에 Two Racks, One Aisle 객체를 통째로 3번 복사하여 6면의 랙을 만든 것은 과잉 설계이므로, 향후 로직 구현 시 단일 랙 병렬배치 기준으로 수정 적용합니다.)*

## 2. C급 스테이션 '예외' 규칙 (거대 존 방식)

*   **명세서 정의:** "C급 스테이션 1개 (SKU 580개 공급)"
*   **시각적 정의:** 도면상 선반(Rack) 4동, 통로 3개, 노동자 다수(4명 이상) 파악됨.
*   **해석 및 규칙:** C급 스테이션은 S/A/B급처럼 랙 1동이 1개의 스테이션 역할을 하는 것이 **아닙니다**. 워낙 많은 SKU를 취급하기 때문에 물리적인 **선반 4동 전체 묶음(거대 창고 존) 자체가 논리적으로 단 1개의 '스테이션'**으로 취급됩니다. 
*   **로직 구현 가이드:** 작업자들은 특정 통로에 고정되지 않고 존 전체를 카트로 순회하므로, AnyLogic 로직 설계 시 C급 구역은 거대한 단일 딜레이(Delay) 구역으로 퉁쳐서 모델링합니다.

## 3. 작업자(Worker) 할당 규칙
*   **S, A, B급:** 각 통로(Aisle)마다 작업자가 고정 배치되어 DPS 방식으로 피킹함.
*   **C급:** 구역 내 작업자 수(4명)가 유동적으로 변동 가능하며, 로봇 대기 시간 공식 `tp = 2.1*qty + 1.3*sku + 24.6(초)`를 연산하는 리소스 풀(Resource Pool)로만 추상화하여 운영함. 물리적 보행(Pedestrian) 모델링은 생략함.
